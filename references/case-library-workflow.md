# PPT 审美案例库工作流

## 为什么不能只用文件夹

文件夹只能保存图片，不能高效复用审美经验。PPT 审美训练需要在 3 秒内按颜色、版式、行业、风格、用途和训练状态找到案例，所以要用 Eagle / Billfish 这类素材管理工具。

第一版 skill 不自动读写 Eagle / Billfish 数据库，只输出采集、标签、取色、复刻和归档 SOP。

## Eagle / Billfish 采集动作

推荐动作：

1. 看到优秀 PPT 单页、海报、网页视觉或设计截图，直接拖拽到 Eagle / Billfish。
2. 使用浏览器插件一键保存网页图片。
3. 保存原图、来源链接、截图日期和简短理由。
4. 自动提取配色，记录主色、辅色、点缀色。
5. 打标签，不要只按日期或项目分类。
6. 复刻完成后，把自己的复刻截图也归档，并标记训练状态。

案例来源：

- 跟我学个P 公众号和课程案例。
- B 站、视频号、站酷、花瓣、优秀发布会和商业报告截图。
- 竞品官网、路演 deck、行业白皮书中的 PPT 化页面。

## 以色找图

当用户有目标配色或模板色板时，用颜色反查案例：

1. 在 Eagle / Billfish 侧边栏点击颜色筛选。
2. 选择目标主色或点缀色。
3. 限定行业、页面用途或风格标签。
4. 找到 3-5 张同色系案例。
5. 比较它们的主色面积、点缀色用途、背景色和素材类型。

输出时不要只说“用蓝色科技风”，要写：

```text
参考色：
检索标签：
候选案例：
可迁移配色：
不可迁移风险：
```

## 自动提取配色

使用工具自动提取 Hex / RGB，减少手动吸管误差。拆参考图时至少记录：

- 主色：大面积背景、标题或核心图形的颜色。
- 辅色：辅助区块、卡片、弱信息背景的颜色。
- 点缀色：按钮、数据高亮、重点文字、小图形的颜色。
- 背景色：页面底色或主背景材质色。
- 取色来源：哪张图、哪一块区域、是否来自模板原色。

不要凭感觉写“蓝紫色”“高级灰”，要尽量保留色值或可复查描述。

## 标签体系

建议标签至少覆盖 6 组：

| 标签组 | 示例 |
|---|---|
| 行业 | 医疗、金融、科技、教育、地产、政务、消费 |
| 页面用途 | 封面、目录、章节页、数据页、流程页、对比页、总结页 |
| 信息关系 | 并列、递进、因果、时间、对比、循环、系统构成 |
| 版式 | 三栏、卡片组、扇形、环绕、中心辐射、左右分栏、上下分区 |
| 风格 | 清爽、科技、新中式、商务、极简、杂志感 |
| 训练状态 | 未拆、已拆、已画 wireframe、已 1:1 复刻、已换内容迁移 |

可以追加颜色标签：蓝色系、绿色系、金色点缀、深色背景、浅色背景。

## 训练作业产物

每次训练必须产出真实文件或截图，不允许只写心得：

- 原参考图路径。
- 四维拆解笔记。
- wireframe 截图。
- 1:1 复刻页截图。
- 换内容迁移页截图。
- 案例卡归档状态。

训练顺序：

1. 先选 1 张优秀页面，不要一口气选太多。
2. 只看 3 秒，说出第一眼机制。
3. 四维拆解，至少写出 2 个具体设计动作。
4. 遮盖文字和图片，画灰色色块 wireframe。
5. 在 PPT 中 1:1 复刻。
6. 换成自己的内容，保留骨架和 Design Spec。
7. 把原图、wireframe、复刻页、迁移页放回案例库。

## 同内容三版式练习

当用户要练版式时，同一组内容至少尝试 3 种骨架：

- 卡片型：适合并列要点、模块化说明。
- 扇形：适合围绕核心对象展开的并列或递进。
- 环绕型：适合中心概念、多角色、多模块生态。

输出时必须说明每种版式适合的内容关系，不要只给视觉形状。

## 未知风格反查

不知道风格名称时：

1. 用百度识图或同类图像反查工具上传参考图。
2. 从搜索结果标题中提取设计关键词。
3. 到花瓣 / 站酷 / 搜索引擎继续搜同类风格。
4. 临时命名该风格，写入案例卡。

输出模板：

```text
临时风格名：
反查关键词：
可用搜索词：
相似参考平台：
需要提取的变量：
```

## 案例库验收

合格案例库不是“有很多图片”，而是可以被调用：

- 能按颜色或标签在 3 秒内找到目标案例。
- 每张已练案例都有四维拆解。
- 每张已练案例都有 wireframe。
- 至少有一张 1:1 复刻截图。
- 至少有一张换内容迁移截图。
- 案例卡写明下一次适合用于什么页面。

如果用户只收藏未练习，必须指出“审美输入还没有转成审美能力”。
