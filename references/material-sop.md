# PPT 素材准入与改造 SOP

## 素材定义

素材不是狭义图片。所有放进 PPT 的元素都算素材：

- 图片；
- 字体；
- 形状；
- 图标；
- 图表；
- 背景；
- 纹理；
- 装饰线条；
- 文件纸张、奖杯、地图、人物、产品等隐喻元素。

判断素材时，不问“它高清吗”，先问“它服务信息吗”。

## 素材准入五问

每个素材进入页面前必须通过五问：

1. 是否直接关联当前页核心信息或数据结论？
2. 是否填补版面结构空白或清晰划分内容区块？
3. 是否引导阅读视线或强化信息层级？
4. 插入后是否造成文字拥挤、背景杂乱或视觉失焦？
5. 是否具备可编辑性，或可由 PPT 原生元素替代？

判断规则：

- 1-3 任一为是，且 4 为否：可保留或改造。
- 1-3 全否：删除。
- 4 为是：压低、替换或删除。
- 5 为否：优先替换为 PPT 原生形状、图表、图标或可控背景。

## 四种结论

| 结论 | 条件 | 动作 |
|---|---|---|
| 保留 | 既辅助信息，又提升美感 | 纳入版式并统一风格 |
| 压低 | 能提升美感，但抢主体 | 降低透明度、放底层、加遮罩 |
| 改造 | 方向对，但尺寸、色调、边缘或风格不匹配 | 调色、裁剪、柔化、渐变、重填色 |
| 删除 | 既不辅助信息，也不提升美感 | 删除，不用装饰弥补 |

不要因为素材高清、贵、稀有、看起来科技，就自动保留。

## 图形优先原则

能用 PPT 原生对象表达的，优先用原生对象：

- 内容容器：矩形 / 圆角矩形。
- 分隔：线条 / 色块 / 留白。
- 并列要点：统一风格图标 + 短标签。
- 数据表达：PPT 原生图表。
- 背景过渡：渐变矩形 / 半透明遮罩。

好处：

- 颜色、大小、透明度可控；
- 后续可编辑；
- 更容易保持风格统一；
- 不依赖不可控网图。

## 图片尺寸无损延展

图片尺寸不足禁止硬拉伸。

适用场景：

- 封面背景图高度不够；
- 章节页主视觉宽度不够；
- 城市、天空、科技背景边缘不够铺满；
- 图片裁剪后局部露白。

操作：

1. 保持图片原比例，不拉伸。
2. 在不足区域覆盖同色系线性渐变矩形。
3. 渐变透明度设为 `0%-100%`，从图片边缘自然过渡到背景色。
4. 必要时加一层浅色或深色半透明遮罩统一文字区域。
5. 复查边缘是否看不出断点。

输出时写成：

```text
不要拉伸图片。
用同色系渐变矩形覆盖图片边缘，透明度 0%-100%，让图片自然延展。
```

## 多素材融合

适用场景：

- 交通大脑封面：城市图 + 云彩图。
- 药品年终总结：针头 + 基因。
- 科技封面：产品图 + 光感背景 + 数据线条。
- 医疗科研：显微图 + 器械 + 分子结构。

操作：

1. 分图下载，不强求一张现成图同时满足所有元素。
2. 按主体、背景、辅助纹理分层。
3. 主体保持清晰，辅助元素降低透明度。
4. 边缘生硬处使用柔化边缘 `15-25pt`。
5. 辅助元素透明度常用 `40%-60%`。
6. 用渐变遮罩或半透明矩形统一色调。
7. 图层叠加后置于底层，不抢正文。
8. 最后用整体色温、饱和度、亮度拉齐。

复查：

- 是否有明显拼接边？
- 是否有素材抢主标题？
- 是否颜色不属于同一个世界？
- 是否文字区域足够干净？

## 图文分区

当页面文字密集、结构模糊时，素材问题常常是结构问题。

操作：

- 用色块划分标题 / 正文。
- 用卡片承载并列要点。
- 用图标建立视觉锚点。
- 并列要点超过 3 项时，转成“统一风格图标 + 提炼短语”。
- 删除只占空间、不建立结构的背景图。

文件类页面可以用：

- 矩形分区；
- 统一图标；
- 文件纸张素材；
- 别针、夹子、页签等轻量隐喻。

不要为了“建筑信息化”强插高楼图；如果高楼图不辅助信息，也不提升美感，就删除。

## 图标改色

图标必须统一线性 / 面性、单色 / 多色、线宽和圆角。

单色图标需要多色时：

1. 优先找 SVG / PPT 可编辑图标。
2. 在 PPT 中取消组合。
3. 拆分路径后重新填色。
4. 所有同级图标使用同一色板。
5. 复查线宽、圆角端点和视觉重量。

不要混用多个网站下载的图标，不要让一个图标比其他图标重很多。

## 图表替换

先判断数据关系，再选图表。

### 多组数据对比

优先条形图 / 柱状图：

- 多个百分比横向对比；
- 多组类别比较；
- 排名；
- 差异大小需要一眼看出来。

不要滥用圆环图 / 环形图。多个圆环并排会让读者计算困难。

### 单一占比

环形图 / 圆环图只在这些条件同时成立时使用：

- 只有一个核心占比；
- 页面留白充足；
- 环体加粗；
- 数据标签外置或放在中心；
- 不需要比较多个值的细小差异。

### 图表复查

- 读者是否需要自己算？
- 最大值和最小值是否一眼可见？
- 标签是否外置且不拥挤？
- 图表颜色是否遵守 Design Spec？
- 图表是否为 PPT 原生可编辑对象？

## 图片调色统一

找不到完全统一的图片时，用 PPT 内置参数强制拉齐：

- 饱和度；
- 亮度；
- 对比度；
- 色温；
- 透明度；
- 裁剪比例。

同一页面或同一套 PPT 的图片不能一张偏冷、一张偏暖、一张高饱和、一张低饱和，除非它们承担明确对比关系。

## 常见错误

- 盲目堆砌：为“不枯燥”插入与内容无关的高清图。
- 图表误用：对比型数据滥用圆环图 / 饼图。
- 尺寸硬拉：图片比例被拉坏。
- 依赖下载：执着寻找 100% 完美素材，不会分图改造。
- 边缘生硬：多图叠加未做柔化边缘、透明度或渐变过渡。
- 风格混乱：实物图、插画、3D、二次元混在一起。
- 不可编辑：关键图表、文字和图标都变成位图。
